# V0.2 First Commit Note

## 提交标题建议

```text
Implement V0.2 visualization baseline and test framework
```

## 提交说明

本次提交完成 USV Sim 项目 **V0.2 的首批核心研发能力建设**，使项目从 V0.1 的“最小可运行仿真内核”升级为“可观察、可演示、可验证”的研究原型平台。

### 本次提交包含

- 增加最小可用的 **2D 实时可视化**
  - `AttackDefenseEnv` 支持 `render_mode="human"`
  - 新增 `Simple2DRenderer`
  - 可显示 attacker、defender、obstacle、goal、boundary

- 增加 **attacker 目标航行 baseline policy**
  - 新增 `GoalSeekingAttackerPolicy`
  - 主接口为 `act(obs)`
  - 支持固定验证场景下的可复现实验与可视化演示

- 增加 **固定 baseline 验证场景**
  - 新增 `baseline_validation` 场景
  - 对应配置文件：
    - `configs/v0_2_default.yaml`
    - `configs/v0_2_baseline_validation.yaml`

- 增加 **测试体系升级**
  - 新增 unit tests
  - 新增 integration tests
  - 升级 smoke test
  - 新增本地 demo 入口

- 扩展 **配置系统**
  - 新增 `AttackerBaselineConfig`
  - 保持对 V0.1 配置的兼容

### 本次提交的工程意义

这次提交解决了 V0.1 的三个直接短板：

1. 没有实时可视化，无法直观看到行为是否符合预期
2. 没有 attacker baseline，难以判断任务/奖励设计是否合理
3. 自动化测试过少，后续迭代缺乏冻结契约保护

V0.2 完成后，项目已经具备：

- 可视化调试能力
- 基线策略验证能力
- 更系统的自动化回归能力
